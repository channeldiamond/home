11:52 – Waiting for OSX build to complete

GMod Update 110 tonight

Wednesday, December 15, 2010 by Garry Newman

First of all, last month while I was on holiday it was GMod’s 4th Birthday! Well, kind of. It was 4 years ago since it was released on Steam. Thanks to Emanuel for the picture!

This patch is kind of a big deal, here’s the changes:

Added PHX content pack
Changed default loading screen
sv_client_predict now defaults to 1
Ragdolls now copy skin, blood colour, material
No longer crashes when modelprecache is full (instead refuses to load new models)
Disabled scriptenforcer when playing demos
Fixed ‘out of edicts’ error – won’t create new entities if we’re near the limit
Servers can now define a loading screen for people joining their server (sv_loadingurl)

[Lua] Fixed not being able to turn off player shadows
[Lua] Added GAMEMODE:CalcViewModelView hook
[Lua] Added ViewModel:ViewModelIndex()
[Lua] Lua entities can override regular entities (when named the same)
[Lua] Added Entity:StopParticleEmission()
[Lua] Added Entity::CreateParticleEffect( strName, iAttachmentType, tabControlPoints )
[Lua] Fixed bug in render.RenderView which meant that after enabling ortho mode you couldn’t turn it off again
[Lua] Added GAMEMODE:PlayerCanPickupItem

[TTT] Added first person ragdoll cam.
[TTT] Added ability for Traitors to hang corpses on walls with the Magneto-stick.
[TTT] Added Glock automatic pistol.
[TTT] Added “X is innocent” radio command.
[TTT] Added “Call detective” button to corpses.
[TTT] Added debug messages to TTT entities for mappers (set “developer 2″).
[TTT] Added check that forces sv_alltalk off if it’s on.
[TTT] Changed gun ironsight positions to be lower (disable in settings).
[TTT] Fixed flaregun scorch marks often not showing up.
[TTT] Fixed thrown explosive barrel kills sometimes being attributed to victim instead of thrower.
[TTT] Fixed Fretta map-only vote restarting while in progress.
[TTT] Fixed ttt_damageowner not working well for certain physics-based traps.

The biggest change being the addition of PHX’s content pack. This includes props, materials, thruster models, wheel models and button models. I don’t want to ruin the fun of exploring these things, so I won’t say much more about them. Thanks to PhoeniX-Storms and his team for allowing us to add this content!

The load screen is now a HTML page. This means, sadly, that the walking icons accross the screen are gone. But on the plus side it means that the load screen is now totally customizable to server owners, and if they want walking icons they should be able to do it with some javascript magic (javascript is fired into the page when a file starts downloading etc).

As usual, let me know of any troubles in the comments or by email and I’ll fix them up as soon as possible.

Update 109 Debrief

Wednesday, November 3, 2010 by Garry Newman

It seems that yesterday’s patch went pretty much without a hitch. People are happily uploading their saves, along with screenshots.

The only problem is that people seem to be having trouble with the new scriptenforcer system. The ConVar no longer directly controls script enforcer. You need to set it, then reload the map. So if scriptenforcer isn’t working for you then this is likely the problem.

If there are any other problems, please let me know in the comments!
